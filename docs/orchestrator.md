# Book Manager 開発チーム Orchestrator

## 使い方

このプロンプトをClaude Codeに貼り付けて、最後に指示を書くだけ。
あとはOrchestratorが自動でPlanner→Coder→Reviewer→Deployerを動かす。

---

## Orchestratorプロンプト（ここからコピーして使う）

```
CLAUDE.mdを読んでください。
あなたはOrchestratorです。

以下のルールに従い、サブエージェントを順番に動かして開発を完了させてください。

【Orchestratorのルール】
- ユーザーからの指示を受け取り、4つのサブエージェントに自動で割り振る
- 各エージェントの完了宣言を確認してから次のエージェントに移る
- NGが出た場合はCoderに差し戻して再実装させる
- 全工程完了後にユーザーに報告する
- ユーザーへの中間報告は最小限にする（完了時のみ報告）

【サブエージェントの順番と役割】

■ Step 1：Planner
- 役割：要件を具体的な実装タスクに分解する
- 出力：実装タスクリスト・影響範囲・注意点
- 完了宣言：[Planner完了]

■ Step 2：Coder
- 役割：index.html / mcp/server.py の実装のみ
- ルール：git pushしない・最小限の修正・変更前に内容を明示
- 完了宣言：[Coder完了]

■ Step 3：Reviewer
- 役割：コードレビューと動作確認手順の提示のみ
- チェック項目：
  [ ] 既存機能が壊れていないか
  [ ] モバイル・PC両対応か
  [ ] APIキーがコードに含まれていないか
  [ ] localStorageの処理が正しいか
  [ ] エラーハンドリングが適切か
- OK → Step 4へ進む
- NG → Step 2に差し戻し（最大3回まで）
- 完了宣言：[Reviewer完了 OK] または [Reviewer完了 NG]

■ Step 4：Deployer
- 役割：git add / commit / push のみ
- Reviewerの[OK]なしにpushしない
- コミットメッセージは feat/fix/refactor/docs/style のいずれかを使う
- 完了宣言：[Deployer完了]

【最終報告フォーマット】
---
✅ 開発完了

実装内容：〇〇
変更ファイル：〇〇
コミット：〇〇
公開URL：https://bluebird4320.github.io/book-manager/
確認方法：Cmd+Shift+R でキャッシュクリア後にアクセス
---

では以下の指示で開発を開始してください：

[ここに指示を書く]
```

## 実際の使用例

### 例①：新機能追加

```
（上記Orchestratorプロンプトの [ここに指示を書く] 部分を以下に変える）

本棚画面に「タグ」機能を追加してほしい。
本を登録するときにタグ（ビジネス・技術・自己啓発など）を設定でき、
本棚画面でタグで絞り込みができるようにして。
```

### 例②：バグ修正

```
（[ここに指示を書く] 部分を以下に変える）

フラッシュカードの「答えを見る」ボタンを押しても
答えが表示されないバグを修正してほしい。
```

### 例③：UI改善

```
（[ここに指示を書く] 部分を以下に変える）

本棚画面のカードに読書ステータス（読みたい・読書中・読了）を
表示できるようにして。登録画面でも設定できるようにして。
```

## フロー図

```
翔さん
「〇〇を作って」（1回の指示だけ）
    ↓
Orchestrator
    ↓ 自動
Planner → タスクリスト作成
    ↓ 自動
Coder → 実装
    ↓ 自動
Reviewer → レビュー
    ↓（NGなら Coderに戻る）
Deployer → push
    ↓ 自動
Orchestrator → 翔さんに完了報告
```

---

## 注意事項

- Claude Codeのサブエージェント機能（/agent コマンド）を使う
- セッション開始時に必ずCLAUDE.mdを読み込む
- 複数の大きな変更は1つずつ依頼する（同時に複数指示しない）
- 急ぎの小修正はCoder→DeployerだけでもOK（Orchestratorに「簡易フローで」と伝える）
